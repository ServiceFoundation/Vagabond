Vagrant

API Overview

The included implementation of ThunkServer is a straightforward distributed implementation that makes use of Vagrant. What follows is a brief overview of the basic API.

A Vagrant environment can be initialized as follows:

1: 
2: 
3: 

open Nessos.Vagrant

let vagrant = Vagrant.Initialize(cacheDirectory = "/tmp/vagrant")

Given an arbitrary object, dependencies are resolved like so:

1: 
2: 
3: 

let value = [ Some(fun x -> printfn "%d" x; x + 1) ; None ; Some id ]

let assemblies = vagrant.ComputeObjectDependencies(value, permitCompilation = true)

An assembly can be exported by writing

1: 

let assemblyPackage = vagrant.CreateAssemblyPackage(assembly, includeAssemblyImage = true)

An assembly package contains necessary data to load the specified assembly in a remote process.

Assemblies can be loaded in a remote process like so:

1: 

let response : AssemblyLoadInfo = vagrant.LoadAssemblyPackage assemblyPackage

Communication

Once all required dependencies have been loaded, communication can be established by using the .Pickler property found in both the server and client instances. These give an instance of FsPickler that is capable of serializing and deserializing objects depending on dynamic assemblies.

1: 
2: 
3: 

let pickler = vagrant.Pickler

let bytes = pickler.Pickle(value)
